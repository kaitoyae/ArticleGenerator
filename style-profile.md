# スタイルプロファイル

## 概要
インタビュー／対話ベースの記事を、読み物として滑らかに再構成する文体。  
「個人の経験」から入り、「社会的な意味」へ接続し、最後は読者の共感や参加意識に着地させる。

## トーン
- 親しみやすいが軽すぎない
- 熱量はあるが、煽情的にはしない
- 相手への敬意を保ちつつ、読者にも開かれている
- まじめなテーマでも、語り口はやわらかい

## 基本構成
1. 導入で人物・テーマ・問題意識をすぐ提示
2. 冒頭で「なぜこの話が重要か」を短く置く
3. 本文は「経験談 → 気づき → 社会的文脈」の順で展開
4. 適宜、問いや要約を挟んで読者を置いていかない
5. 終盤は視座を広げ、未来志向・参加余地で締める

## 文体の特徴
- 一文は比較的短め、2〜4文で段落を切る
- 口語ベースだが、崩しすぎない
- 接続は自然に。「そこで」「一方で」「実は」「たとえば」を多用
- 抽象論だけで進めず、具体例を先に置く
- 強い断定は避け、「〜かもしれない」「〜ように感じる」で余白を残す
- 読者の疑問を先回りして、小さな問いを差し込む

## 見出しの傾向
- 説明的すぎず、内容がひと目でわかる
- 会話のエッセンスを要約した見出し
- やや感情や問題意識を含めてもよい
- 長すぎない

## 内容運びの型
- 個人の原体験をフックにする
- その体験を、制度・教育・社会課題へ接続する
- 1トピックごとに「具体」と「抽象」を往復する
- 読後に「自分にも関係がある」と感じさせる

## 表現ルール
- 専門用語は必要最小限。使う場合はすぐ言い換える
- 誇張表現は控えめ
- 相手の発言は要約しつつ、温度感を残す
- 評価よりも理解を優先する
- ネガティブな話題でも、閉じずに希望や可能性へ戻す

## AI執筆時の再現指針
- まず「誰の、どんな経験の話か」を冒頭で明示
- 単なるQ&A列挙にせず、編集後記のように文脈をつなぐ
- 各セクションに最低1つ、具体エピソードを入れる
- 社会的意義は毎段落で重く語らず、後半で効かせる
- 最後は結論で閉じ切らず、読者に考える余地を残す

## 避けること
- 断定しすぎる評論調
- 情報を詰め込みすぎた長文
- 感情過多な美談化
- 専門性を見せるための難解表現
- ただの逐語録のような構成
